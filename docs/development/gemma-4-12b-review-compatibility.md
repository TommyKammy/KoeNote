# Gemma 4 12B review model compatibility

Last verified: 2026-06-14.

This note captures the model compatibility decision for replacing the
`llm-jp-4-8b-thinking-q4-k-m` review model candidate with Gemma 4 12B QAT.
It is intentionally limited to the pre-catalog verification work tracked by
GitHub issue #61.

## Decision

Use the official GGUF repository for KoeNote's current review runtime:

- Repository: <https://huggingface.co/google/gemma-4-12B-it-qat-q4_0-gguf>
- Model file: `gemma-4-12b-it-qat-q4_0.gguf`
- License: Apache-2.0
- Proposed KoeNote model id: `gemma-4-12b-it-qat-q4-0`

Do not catalog the unquantized QAT repository directly:

- Repository: <https://huggingface.co/google/gemma-4-12B-it-qat-q4_0-unquantized>
- Format: Transformers / Safetensors

KoeNote's review model path currently uses `llama.cpp` with a GGUF model file.
The unquantized repository is useful for Transformers-based research or custom
compilation, but it is not the direct download target for the existing
`llama-completion.exe` review path.

## File selection

The GGUF repository currently exposes these deployable files:

- `gemma-4-12b-it-qat-q4_0.gguf`
- `mmproj-gemma-4-12b-it-qat-q4_0.gguf`

For the current text-only review path, catalog only
`gemma-4-12b-it-qat-q4_0.gguf`. The `mmproj` file is for multimodal projection
support and should not be added to the review model catalog until KoeNote has a
separate design for multimodal review inputs.

## Local runtime compatibility

The bundled runtime at `tools/review/llama-completion.exe` supports the options
needed for this model family:

- `-m, --model` for a local GGUF file
- `--hf-repo` and `--hf-file` for Hugging Face smoke checks
- `--no-conversation` for KoeNote's current raw-prompt review path
- `--chat-template gemma` for optional chat-template probes outside the current
  app-equivalent review path
- `--reasoning on|off|auto`
- CUDA backend loading when the bundled CUDA DLLs are present

The local device probe on 2026-06-14 reported:

```text
CUDA0: NVIDIA GeForce RTX 3090 (24575 MiB, 23332 MiB free)
```

That confirms the review runtime can see the local CUDA backend. It does not, by
itself, prove the 12B GGUF fits every target machine.

## Smoke check

After the GGUF file is downloaded, use this command to verify that the bundled
runtime can load it through the same no-conversation path that KoeNote uses for
review jobs. The managed download path includes the role and model id directory
created by `ModelInstallService.GetDefaultInstallPath`.

```powershell
$StorageRoot = "<KoeNote model storage root>"
$Model = Join-Path $StorageRoot "review\gemma-4-12b-it-qat-q4-0\gemma-4-12b-it-qat-q4_0.gguf"
$Prompt = ".\tmp\review.prompt.txt"
$Schema = ".\tmp\review.schema.json"
.\tools\review\llama-completion.exe `
  --model $Model `
  --file $Prompt `
  --ctx-size 8192 `
  --n-gpu-layers 999 `
  --n-predict 4096 `
  --temp 0.1 `
  --no-conversation `
  --no-display-prompt `
  --json-schema-file $Schema
```

Set `$StorageRoot` to the model storage root selected during setup. The default
current-user root is `%LOCALAPPDATA%\KoeNote\models`, while an all-users install
can use `%ProgramData%\KoeNote\models`; custom setup storage roots should use
their configured value. Use a UTF-8 prompt file generated by a real Review job,
or a small hand-written Review prompt that asks for the same JSON
correction-draft shape. Use the generated `review.schema.json` when validating
the JSON-schema Review path.

If the model is not downloaded locally and a network smoke is acceptable, the
same runtime can fetch the GGUF from Hugging Face. This confirms the runtime can
resolve the target file, but it is not a substitute for the managed-path command
above:

```powershell
$Prompt = ".\tmp\review.prompt.txt"
.\tools\review\llama-completion.exe `
  --hf-repo google/gemma-4-12B-it-qat-q4_0-gguf:Q4_0 `
  --hf-file gemma-4-12b-it-qat-q4_0.gguf `
  --no-mmproj `
  --file $Prompt `
  --ctx-size 8192 `
  --n-gpu-layers 999 `
  --n-predict 4096 `
  --temp 0.1 `
  --no-conversation `
  --no-display-prompt
```

Do not rely on a `--chat-template gemma` / `--single-turn` probe as the only
compatibility signal unless #62 or a later runtime change moves KoeNote's Review
path to chat-template mode.

## Catalog-era validation

Issue #64 records the post-catalog release decision. Use the repository smoke
script for repeatable checks:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\development\Test-Gemma4ReviewSmoke.ps1
```

The script verifies:

- the built-in catalog contains `gemma-4-12b-it-qat-q4-0`
- only the `experimental` preset selects the 12B model
- `recommended` and `high_accuracy` remain on the safer E4B model
- the Hugging Face `resolve/main` URL returns the expected HEAD metadata
- a one-byte range request succeeds, proving the artifact is reachable without

- `tools\review\llama-completion.exe` exposes the required HF, JSON schema,
  no-conversation, reasoning, and chat-template options

```powershell
  -RunLocalRuntimeSmoke `

```

The local smoke is intentionally opt-in because the model is large. It should be
run on release candidate hardware before considering any promotion beyond the
experimental preset.

## Release decision

Ship Gemma 4 12B QAT GGUF as `experimental` only.

Do not promote it to `recommended` or `high_accuracy` until full local
load/generation smoke and field data show that its memory use and initial load
time are acceptable compared with `gemma-4-e4b-it-q4-k-m`.

On this host, `tools\review\llama-completion.exe` and CUDA backend discovery are
available, but the full `gemma-4-12b-it-qat-q4_0.gguf` file was not present in
the standard KoeNote model storage roots. The issue #64 release judgement is
therefore: keep the model cataloged and selectable through `experimental`, but
do not make it a default.

## Catalog implementation record

The catalog implementation was handled in issue #62 with these guardrails:

- A new model id, `gemma-4-12b-it-qat-q4-0`, was added.
- `llm-jp-4-8b-thinking-q4-k-m` remains available for compatibility.
- Only the `experimental` preset points at the 12B model.
- `recommended` and `high_accuracy` stay on `gemma-4-e4b-it-q4-k-m` until the
  heavier 12B model has enough field validation.
- Fallback and existing install migration were kept separate under issue #63.
