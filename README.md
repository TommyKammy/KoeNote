  KoeNote

日本語の音声ファイルを、ローカルPC上で文字起こし・推敲レビュー・編集・出力できる Windows アプリです。

![KoeNote main screen](docs/images/koenote-main.png)

## KoeNote でできること

- 音声ファイルを登録して文字起こしできます
- 文字起こし結果を確認しながら音声を再生できます
- 推敲レビューで、より自然な文章への修正候補を確認できます
- 話者名や文字起こし本文をインライン編集できます
- TXT / JSON / SRT / DOCX 形式で出力できます
- 音声データや文字起こしデータは、基本的に利用者のPC内で扱います
- GitHub Releases と `latest.json` を使った更新確認に対応しています

## 必要な環境

### 対応OS

- Windows 10 / Windows 11
- x64 環境

### ハードウェア要件

KoeNote はローカルPC上で音声変換、ASR、推敲レビューを行うため、音声の長さや利用モデルによって必要な性能が変わります。

| 項目 | 最小目安 | 推奨 |
| --- | --- | --- |
| CPU | 4コア以上 | 8コア以上 |
| メモリ | 16GB | 32GB以上 |
| ストレージ | 20GB以上の空き容量 | 50GB以上の空き容量 |
| GPU | 任意 | NVIDIA GPU 8GB VRAM以上 |

GPUは必須ではありませんが、対応環境では文字起こしや推敲処理が速くなる場合があります。モデルファイルは容量が大きいため、初回セットアップ前に十分な空き容量を確保してください。

### 追加で必要になるもの

- 音声変換に使うランタイム
- ASRモデル
- 推敲レビュー用LLM

これらはアプリ内のセットアップ画面やモデル管理画面から確認・導入できます。

## セットアップ

初回利用時は、ヘッダー右側のセットアップアイコンからセットアップ画面を開きます。

セットアップでは、主に次の項目を確認します。

- 保存先
- 実行環境
- ASRモデル
- 推敲レビュー用LLM
- ライセンス確認
- モデル導入状態
- 簡易動作確認

### 1. 保存先を確認する

KoeNote のジョブ、ログ、モデル関連データを保存する場所を確認します。モデルは容量が大きくなるため、空き容量に余裕のあるドライブを選ぶことをおすすめします。

### 2. モデルを準備する

文字起こしにはASRモデル、推敲レビューにはLLMが必要です。セットアップ画面の案内に従って、推奨モデルを導入するか、すでにPC内にあるモデルを登録してください。

推敲レビューを使わない場合は、ヘッダーの **推敲 ON / OFF** で推敲ステージをOFFにできます。推敲をOFFにすると、ASRモデルだけで文字起こしを実行できます。

### 3. ライセンスを確認する

利用するモデルや外部ランタイムには、それぞれ個別のライセンスがあります。セットアップ中に表示されるライセンス情報を確認してください。

### 4. 動作確認を行う

セットアップが完了したら、短い音声ファイルで文字起こしを試してください。実行できない場合は、ログ画面でエラー内容を確認できます。

## はじめて使う流れ

1. KoeNote を起動します
2. 左側のジョブ欄下部にあるインポート枠へ音声ファイルをドラッグ＆ドロップします
3. クリックしてファイル選択から音声ファイルを追加することもできます
4. ジョブ一覧に音声が登録されたら **実行** を押します
5. 文字起こしが完了したら、中央の文字起こし欄で内容を確認します
6. 必要に応じて、話者名や本文をクリックして編集します
7. 必要に応じて、右側の推敲レビュー候補を適用または却下します
8. **エクスポート** メニューから必要な形式を選んで出力します

## 画面の見方

### ヘッダー

画面上部には、よく使う操作があります。

- **実行**: 選択中のジョブを処理します
- **中止**: 実行中の処理を止めます
- **推敲 ON / OFF**: 推敲レビューステージを切り替えます
- **エクスポート**: TXT / JSON / SRT / DOCX 出力、タイムスタンプ設定、出力フォルダーを開く操作を行います
- **VERSION**: 現在利用中のKoeNoteバージョンを表示します
- **ASR / REVIEW**: 利用中の文字起こしモデルと推敲モデルを表示します
- **設定 / セットアップ / モデル / ログ / 更新確認**: 環境やモデル、ログ、アップデートを確認します

### ジョブ

左側には、登録した音声ファイルが表示されます。

- 登録済み、実行中、レビュー待ち、レビュー完了などの状態を確認できます
- ジョブごとに進捗率が表示されます
- 不要なジョブはクリア履歴へ移動できます
- ジョブ欄下部のインポート枠から音声ファイルを追加できます

### 文字起こし

中央には、文字起こし結果が表示されます。

- 音声の再生位置に合わせて内容を確認できます
- 本文は編集したいセグメントをダブルクリックして編集できます
- 話者名はクリックして編集できます
- 別の場所をクリックすると編集内容が自動保存されます
- 自動スクロールを有効にすると、再生位置に近いセグメントへ追従します

### 推敲レビュー

右側には、推敲候補が表示されます。

- ASR結果が十分自然な場合、候補は表示されないことがあります
- 候補を適用または却下しながら本文を整えられます
- 推敲ステージをOFFにすると、推敲レビューをスキップできます

### 処理ステージと再生バー

画面下部には、処理状況と音声再生バーがあります。

- **音声変換**
- **ASR**
- **推敲**
- **レビュー**

各ステージの状態を見ながら、音声を再生・停止・シークできます。

## モデル管理

ヘッダーのモデルアイコンから、文字起こしや推敲に使うモデルを確認できます。

- インストール済みモデルの確認
- 使用するモデルの切り替え
- モデルファイルの削除
- ライセンス確認

モデルファイルは容量が大きい場合があります。不要なモデルは削除することで、PCの空き容量を増やせます。

## データの保存場所

KoeNote のデータは、主に以下の場所に保存されます。

```text
%APPDATA%\KoeNote
%LOCALAPPDATA%\KoeNote
```

ログはアプリ内のログ画面から確認できます。

## 出力形式

KoeNote は以下の形式で出力できます。

- TXT
- JSON
- SRT
- DOCX

字幕用途では SRT、文章として確認・共有する場合は TXT または DOCX が便利です。

## アップデート

KoeNote はGitHub Releasesで配布される更新情報を確認できます。ヘッダーの更新確認アイコンから、現在のバージョンが最新版かどうかを確認できます。

ホビープロジェクトとして配布しているため、現在のMSIは未署名です。インストール時に Windows Defender SmartScreen の警告が表示される場合があります。
